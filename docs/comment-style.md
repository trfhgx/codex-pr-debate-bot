# PR Comment Style

Write comments that are concrete, scoped, and easy to answer.

- Ask 2-3 questions when human input is needed.
- Put the recommended answer directly under each question.
- If code evidence resolves a branch of the design tree, say what file or API resolved it.
- Do not claim implementation started until the codex-lb job has been accepted.
- Final implementation comments should include changed behavior, files touched, tests run, pushed branch/commit if known, and blockers.
